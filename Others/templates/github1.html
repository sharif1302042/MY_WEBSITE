Git Version Control

ডেভেলপমেন্ট এর জগতে গুরুত্বপূর্ণ যে কয়টি টুল আছে তার মধ্যে গিট হচ্ছে অন্যতম | আমার মতে এটি সবচেয়ে ইউজফুল টুল |
আমরা সাধারণত যা করি, আমাদের প্রোজেক্ট এ কোন নতুন ফিচার অ্যাড করি, তারপর তা টেস্ট করি | টেস্ট করার পর যদি মনে হয় ঠিক আছে তাহলে সেই ফিচার টা রেখে দেই আর যদি মনে হয় না আগের টাই ভাল ছিল তখন আবার আগের টাতে ফিরে যাই , কোথায় কোথায় চেঞ্জ করেছিলাম তা খুঁজে বের করে তা আগের মত করে সাজাই |
যদি অনেক বড় প্রজেক্ট হয় তাহলে ঠেলাটা বুঝুন কেমন লাগবে ??? এভাবে কাজ করাটা কতটা বিরক্তি কর আর সময়সাপেক্ষ ??? তখন কিন্তু মনে হওয়াটা স্বাভাবিক যদি একটা আলাদীনের চেরাগ পাইতাম যা আমাকে টাইম মেশিনে করে আগের কাজটা ফিরায় দিবে |
ঠিক এই কাজটাই গিট খুব সহজেই করে দেয় | শুধু এটাই নয় গিট আপনার প্রতিটা কাজ ট্রাক করে রাখবে | কোথায় চেঞ্জ করেছিলেন তার প্রতিটা চেঞ্জ আলাদাভাবে ট্রাক করে রাখবে এবং আপনার প্রতিটা কাজের ট্রাক সেভ করে রাখে| আপনি চাইলেই আপনার পূর্বের ফিচার এ ফিরে যেতে পারবেন | অনেক বকর বকর করলাম নিজের খেয়ে গিট এর সাফাই গাইলাম এইবারর কাজের কাজে আসি .....

গিট ইন্সটল করিঃ

For Ubuntu $ sudo apt install git
For Fedora $ sudo dnf install git

নিচের কমান্ড দিয়ে ভার্সন চেক করুন

$ git --version

গিট কনফিগার করিঃ

নিচের দুইটা পদ্বতির মাধ্যমে গিট কনফিগার করা যায় |

$ git config --global user.name "User_name"
$ git config --global user.email "User_Email"

$ git config user.name "User_name"
$ git config user.email "User_Email"

পার্থক্যটা হয়ত ধরতে পেরেছেন | সমস্যাটা --global কে নিয়ে ধরুন আপনি আপনার সকল প্রোজেক্ট এর জন্য একটা গিট ইউজ করতে চাচ্ছেন তাহলে এই --global কিওয়ার্ড ইউজ করতে হবে আর যদি চান যে না আমি প্রত্যেকটা প্রোজেক্টে আলাদা আলাদা গিট ইউজ করবো তাহলে ২নং টা ইউজ করতে হবে |

গিট রিপোজিটরি(রিপো) তৈরি করুনঃ

গিটে ডিরেক্টরিকেই রিপোজিটরি সংক্ষেপে রিপো বলে | যেই ডিরেক্টরিতে আপনার প্রোজেক্ট টি আছে সেখানে যান | আমার প্রোজেক্ট project নামে একটি ডিরেক্টরি আছে এবং এটাতে দুটো ফাইল আছে my_file.txt ও index.html নামে এটাতে আমি গিট ইউজ করতে চাচ্ছি |
গিটে ডিরেক্টরিকেই রিপোজিটরি সংক্ষেপে রিপো বলে | যেই ডিরেক্টরিতে আপনার প্রোজেক্ট টি আছে সেখানে যান | আমার প্রোজেক্ট project নামে একটি ডিরেক্টরি আছে এটাতে আমি গিট ইউজ করতে চাচ্ছি | এর জন্য নিচের কমান্ড টি দিয়ে গিট কে জানিয়ে দিন হে গিট বাবা আমি এই ডিরেক্টরি আপনার নামে সপে দিলাম

$ cd project
$ git init

কমান্ড টি দেয়ার পর নিচের মত আউটপুট পাই |

বলতে পারি আমার ডিরেক্টরি গিটের নামে সপে দেয়া হয়েছে অর্থাৎ এই ডিরেক্টরির ভিতরে আপাতত যা কিছু আছে গিট সবকিছু ট্রাক করতে পারবে |

বলে রাখি, গিট আপনার সবকিছু কিন্তু এখনি ভার্সন হিসাবে সেভ করবে না এর জন্য আর একটু কষ্ট করা লাগবে | আস্তে আস্তে সামনের দিকে যান ......

গিটের স্ট্যাটাস দেখিঃ

গিটের বর্তমান অবস্থা জানার জন্য নিচের কমান্ড দিন

$ git status

আর এটা হচ্ছে আমার গিটের বর্তমান অবস্থা |

গিটের সাথে আমার ডিরেক্টরি কে পরিচয় করিয়ে দেয়া হয়েছে ঠিকি কিন্তু কোন কোন ফাইলকে সে ট্রাক করবে তা কিন্তু গিটকে জানানো হয়নি, সে এই কাজটা কিন্তু নিজে নিজে করবেনা , তাকে জানিয়ে দিতে হবে গিট বাবা তুমি আমার এই এই ফাইল কে ট্রাক কর অথবা সবগুলাই ট্রাক কর |
একটু খেয়াল করে দেখুন গিট কিন্তু আমাদের কিছু সাজেস্ট করতেছে ফাইলে অ্যাড করার জন্য এই এই কমান্ড দিন | তো চলুন দেরি কেন অ্যাড করে দেই ...

শুরু করুনঃ

যেহেতু আমার ডিরেক্টরি তে দুইটা ফাইল আছে আমি চাচ্ছি আপাতত গিট index.html ফাইলকে ট্রাক করুক | পরে যদি প্রয়োজন হয় অন্য ফাইল গুলকেও ট্রাক করতে বলবো | এর জন্য নিচের কমান্ড দিন

$ git add index.html

যেহেতু আমি কে ট্রাক করতে চাচ্ছি তাই ফাইলনেম হিসাবে দিয়েছি আপনি যাকে ট্রাক করতে চান তার নাম দিন |
এখন গিট আপনার জন্য কি করলো তা দেখার জন্য গিটের স্ট্যাটাস চেক করুন

$ git status

গিট বাবা আমার জন্য নিচের কাজটি করে দিয়েছে |

খেয়াল করুন এখানে দুইটা পার্ট দেখা যাচ্ছে | প্রথমটা Changes to be committed: এবং পরেরটা Untracked file: | আপনি যেই ফাইলটা ট্রাক করতে বলেছেন সেটা প্রথন অংশে আছে এবং পরেরটাতে যেই ফাইলটা ট্রাক করতে বলেননি সেটাকে স্টোর করেছে |
বুঝতেই পারতেছেন গিট ঘটনা ঘটায় দিছে |

এখন আমরা চাচ্ছি গিট ডিরেক্টরিতে রাখা সব ফাইল গুলোকেই ট্রাক করুক, তাহলে নিচের কমান্ড দিন , এরপর আবার গিটের স্ট্যাটাস চেক করুন এবং দেখুন গিট আপনার জন্য কিছু করেছে কিনা

$ git add --all
$ git status

গিট আমার জন্য নিচের কাজটা করেছে

বুঝতেই পারতেছেন ডিরেক্টরি তে যত ফাইল ছিল সব তার আওতায় নিয়ে নিয়েছে |
একটু খেয়াল করুন Changes to be committed: এইরকম কিছু একটা দেখা যাচ্ছে এইটা আবার কি ???

কমিট করাঃ

আপনি যে আপনার ফাইল গুলো গিটে সপে দিয়েছেন এইগুলা তো সবসময় আর এইভাবেই থাকবেনা মাঝে মাঝে এডিট টেডিট করবেন তাই নয় কি??
এখন আপনি কি কি চেঞ্জ করলেন এইটা বোঝার জন্য কমেন্ট করতে হয় | গিট কমিট দিয়ে আপনি কমেন্ট করে বলে রাখেন এই ফাইল টা আপনি এই এই জায়গায় এই এই জন্য এডিট করেছিলেন |
এটাতে আপনারই লাভ, আপনার বুঝতে সুবিধা হবে যে আগের ফাইল টা কি রকম ছিল ,কোথায় চেঞ্জ করেছিলেন, আগের ভার্সন এ ফিরে যেতেও সুবিধা পাবেন |
তো কমিট করে ফেলুন ...।।

$ git commit -m "index.html & my_file.txt added for 1st time"

মুটামুটি বলা যায় বেসিক কাজটুকু করে ফেলেছেন মানে আপনার প্রোজেক্ট টি এখন গিটের উপর সপে দেয়া আছে | আপনি গিটের ফায়দা লুটতে থাকেন |

এই পর্ব এই পর্যন্তই | এর মাধ্যমে আপনি বেসিক কিছু শিখলেন | গিট সম্বন্ধে আরও অনেক জানার আছে | আমার জানামতে সবচেয়ে ভাল রিসোর্স টা পাবেন | গিট এর আডভান্স কিছু টপিক নিয়ে পরে আর একটা টপিক লিখার ইচ্ছা আছে ইনশাআল্লাহ |
